2. SYNTHÈSE PRODUIT
-------------------
  Application web statique de facturation (cible conformité DGI Maroc),
  en français, données locales (navigateur : OPFS / stockage versionné),
  installable en PWA (hors ligne après chargement). Pas de backend obligatoire
  pour l’usage courant. Licence locale ou session d’essai (voir auth).


3. ÉTAT TECHNIQUE (APERÇU)
--------------------------
  Stack        HTML + CSS modulaire + JavaScript « vanilla », sans bundler.
               Node ≥ 18 pour lint, Prettier, Jest, scripts npm.

  Point d’entrée index.html (shell, modales globales) ; contenu des écrans
  injecté via js/page-templates.js (généré par scripts/build-page-templates.mjs).

  Fichiers lourds    ui.js, docs.js — navigation dans le code plus exigeante.

  PWA                manifest.json, sw.js, sw-register.js.
                     Au moment de la rédaction : CACHE_NAME = « invo-v7 »
                     (à incrémenter encore si tu changes le précache ou des
                     assets critiques).

  Identité affichée  manifest : name / short_name orientés « INVOOFFICE » ;
                     index.html : title, application-name, apple-mobile-web-app-title
                     alignés (cohérence titre PWA / onglet / iOS).


4. INTERFACE ET EXPÉRIENCE
--------------------------
  Design system décrit dans docs/design-system.md : tokens (tokens.css),
  composants (boutons, modales, formulaires, toasts, tselect), fichiers
  pages/*.css, responsive dédié.

  Icônes PWA : manifest référence icons/icon-192.png, icon-512.png,
  variantes maskable ; apple-touch-icon et logo shell pointent vers
  icon-512.png. Pas de mesure Lighthouse stockée dans le dépôt.

  Dette UX/a11y : passage significatif sur js/page-templates.js (emojis /
  SVG décoratifs masqués pour AT, type="button", groupes de période, options
  et placeholders sans emoji seul) ; poursuivre si Lighthouse signale encore
  des écarts (tableaux dynamiques, focus).

11. AUDIT EXPERT — VERDICT DÉTAILLÉ (MÊME PÉRIMÈTRE QUE CE DOCUMENT)
--------------------------------------------------------------------
  Produit
    Facturation orientée Maroc (DGI), 100 % client, français, données locales.
    La valeur = autonomie et données chez l’utilisateur ; le modèle impose
    sauvegardes / exports (déjà partiellement dans l’UX).

  Technique
    Vanilla + fichiers statiques : déploiement simple, scripts listés dans
    index.html. Contrepartie : pas de code-splitting auto ; libs lourdes
    (PDF, tableur…) — acceptable sur poste, à surveiller sur mobile faible.

  PWA
    Ensemble manifest + service worker + stratégie de mise à jour adapté au
    produit. Point d’attention opérationnel : discipline CACHE_NAME, précache,
    smoke test après release.

  Données
    Versionnement + migrations dans storage.js : bon réflexe pour la durée.
    Tout changement de schéma doit rester explicite (version + migration testée).

  Interface
    Design system et tokens = base pro. Dette principale : homogénéité entre
    shell et page-templates.js (structure, aria, titres).

  Qualité logicielle
    Lint + tests unitaires sur zones critiques : bon complément à une logique
    métier riche. Prochain palier de confiance : quelques scénarios E2E
    (parcours licence, création document, export).

  Documentation
    README présent : npm, déploiement, checklist release ; maintenir à jour
    quand les scripts ou le flux de release changent.

  Verdict en une phrase
    Niveau « bon outil sérieux » vers produit abouti : ~7,3/10 ; les plus gros
    leviers restants : affinage a11y (Lighthouse), E2E léger, clarté sur les gros fichiers,
    sans remettre en cause la stack actuelle.
